별도 랜딩 페이지를 만들고 기존 메인을 /dashboard로 이동해줘.

## 1. 페이지 구조 변경
- app/page.tsx (뉴스 허브) → app/dashboard/page.tsx 로 이동
- app/ads/page.tsx → app/dashboard/ads/page.tsx 로 이동
- app/page.tsx → 새 랜딩 페이지로 교체

## 2. 새 랜딩 페이지 (app/page.tsx)

### 레이아웃 구조
풀스크린 히어로 섹션 (min-height: 100vh) 안에 아래 순서로 배치:

[상단]
- 서비스명: "2026 게임 트렌드"
- 메인 카피: "글로벌 게임 산업의 뉴스와 광고 트렌드를 한눈에"
- 서브 카피: "AI가 매일 선별하는 게임 업계 핵심 뉴스, Meta 광고 라이브러리 기반 실시간 크리에이티브 트렌드"
- "대시보드 바로가기" 버튼 → /dashboard 이동

[중단 - 숫자 지표 4개 가로 배치]
- 11 / 뉴스 채널 (하드코딩)
- OOO+ / 수집 뉴스 최근 3일 (/api/news/fetch 캐시에서 allNews.length, 로딩 중 "---")
- 2 / AI 분석 섹션 (하드코딩)
- 매일 / 업데이트 (하드코딩)
- 숫자는 카운트업 애니메이션 (0에서 실제 숫자까지)

[하단 - 주요 기능 소개 카드 3개]
- 게임 뉴스: "11개 채널 실시간 수집"
- 광고 트렌드: "Meta 광고 라이브러리 기반"
- AI 분석: "매일 Top 5 자동 선별"
- 카드 호버 시 살짝 올라오는 효과

[최하단]
- "대시보드 바로가기" 버튼 한 번 더

### 디자인 원칙
- 다크/라이트 모드 모두 지원 (기존 CSS 변수 사용)
- 카피 텍스트 크고 임팩트 있게
- accent 색상(레드/핑크) 포인트로 활용
- 미니멀하고 여백 넉넉하게

## 3. 퀵메뉴 (app/layout.tsx) 업데이트
- "홈" → / (랜딩 페이지)
- "대시보드" → /dashboard
- "광고 트렌드" → /dashboard/ads

- 기존 /ads 참조 → /dashboard/ads 로 변경
